# MathForge 배포 가이드

## 1. 다른 Windows PC에서 실행

가장 쉬운 방법입니다.

1. 이 폴더 전체를 압축합니다.
2. 다른 PC에서 압축을 풉니다.
3. Node.js LTS 버전을 설치합니다.
4. `start-mathforge.bat`을 더블클릭합니다.
5. 브라우저에서 `http://127.0.0.1:3100/index.html`이 열립니다.

주의:

- 문제 기록은 각 PC의 브라우저 저장소에 저장됩니다.
- 다른 PC로 기록을 옮기려면 설정 메뉴의 `JSON 백업`과 `JSON 복원`을 사용하세요.
- API Key도 JSON 백업에 포함될 수 있으므로 백업 파일 공유에 주의하세요.

## 2. 같은 네트워크의 다른 기기에서 접속

현재 서버는 보안을 위해 `127.0.0.1`에서만 접근하도록 되어 있습니다.

같은 와이파이나 사내망에서 다른 기기가 접속하게 하려면 `server.mjs`의 host 값을 바꿔야 합니다.

```js
const host = "0.0.0.0";
```

그 뒤 실행 PC의 내부 IP로 접속합니다.

```text
http://실행PC_IP:3100/index.html
```

주의:

- 방화벽에서 3100번 포트를 허용해야 할 수 있습니다.
- API Key를 브라우저에 저장하는 구조라 공용 네트워크 배포에는 적합하지 않습니다.

## 3. 웹사이트로 배포

정적 사이트로 먼저 배포할 수 있습니다.

추천:

- GitHub Pages
- Netlify
- Vercel
- Cloudflare Pages

업로드 대상 파일:

- `index.html`
- `styles.css`
- `app.js`
- 필요하면 `README.md`

AI API를 실제로 붙일 때는 프론트엔드에서 직접 API Key를 쓰지 말고 백엔드를 추가하세요.

권장 구조:

```text
브라우저 앱 -> 백엔드 API -> Gemini/OpenAI API
```

## 4. 데스크탑 앱으로 배포

현재 웹앱을 데스크탑 앱으로 감싸려면 두 가지 선택지가 좋습니다.

### Tauri

- 배포 파일이 작습니다.
- Windows/macOS 앱으로 만들기 좋습니다.
- Rust 기반 빌드 환경이 필요합니다.

### Electron

- 구현 자료가 많습니다.
- 웹앱을 감싸기 쉽습니다.
- 배포 파일이 비교적 큽니다.

추천 순서:

1. 지금은 `start-mathforge.bat`으로 내부 테스트
2. 사용자 피드백 반영
3. Tauri 또는 Electron 패키징
4. 자동 업데이트와 설치 프로그램 추가

## 5. 모바일 앱 또는 설치형 웹앱

초기에는 PWA가 가장 현실적입니다.

- 웹사이트를 설치형 앱처럼 사용할 수 있습니다.
- 앱스토어 심사 없이 배포할 수 있습니다.
- 나중에 React Native 또는 Flutter 앱으로 확장할 수 있습니다.
